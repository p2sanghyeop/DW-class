관계연산자
=> 
>, <, <=, >=, =
!=(다르다)->같지않을때

함수
count(*)=숫자만 확인할때
as'총합' - 별칭을 준다

논리 연산자
and - 두개가 참일때 쓴다 - 조건을 두번 설정
or - 둘중에 하나라도 맞을때
order by - 정렬 - 맨뒤에 온다
desc - 내림차순
asc - 오름차순

해석순서
1. from-> 2. where-> 3. group by-> 4. having
-> 5. select-> 6. order by

null - 해당 데이터에 값을 안넣은 상태 - 되도록 조회하지 말자(회사에서 욕먹는다)
is - 해당 데이터만 조회
is not - 해당 데이터를 제외하고 조회

like - 이름같은것을 조회할때 사용
a% - a로 시작하는 사람조회
%l%l% - l이 두번들어가는 사람 조회
ex)  select
     job,
     ename
     from emp
     where ename like 'a%'

insert into - 어디  테이블에 저장할거니?

ex) insert into emp
    (empno,ename,job,mgr,HIREDATE,sal,DEPTNO)
    values
    (8002, '홍길동','MANAGER', 7782,'2022-01-06', 300, 20)
insert 주의점 - 제대로된 컬럼에 제대로된 데이터를 넣는지 확인

update - 내부적으로 데이터를 delete 하고 insert 한다
set - 특정 데이터를 update할때 사용
ex) update  emp
    set ename = '정우성'
    where empno = 8002;
    
 ex) update  emp
     set 
     sal = 5000,
     job = 'manager'
     where empno = 8002;

count - 갯수
sum - 총합
avg - 평균
max - 최대
min - 최소

group by
ex) select
    job
    from emp
    group by job
having group by를 했을때 필터링
ex) select
    job,
    sum(sal)
    from emp
    group by job
    having sum(sal) > 6000
group by를 할때는 그룹이 되는 컬럼이 와야 한다
